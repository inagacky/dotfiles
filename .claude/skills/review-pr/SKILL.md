# PR コードレビュー

指定されたPull Requestのコードレビューを実施する。
Claude Code自身のレビューに加え、Codex MCP server経由でCodexにもレビューを依頼し、両方の観点を統合した結果を出力する。

## 対象PRの特定

- 引数としてPR番号が指定された場合: `$ARGUMENTS` を対象PRとする
- 引数が省略された場合: 現在のブランチに紐づくPRを `gh pr view` で特定する

## フェーズ1: Claude Codeによるレビュー

### 1-1. PR情報の取得

1. `gh pr view <PR番号> --json title,body,baseRefName,headRefName` でPR概要を取得する
2. `gh pr diff <PR番号>` でPRの差分を取得する
3. 差分が大きい場合は `gh pr view <PR番号> --json files` で変更ファイル一覧を確認し、ファイルごとに内容を確認する

### 1-2. レビュー観点

以下の観点で変更内容を評価する:

#### コード品質
- コーディング規約・スタイルの一貫性
- 可読性（命名規則、関数の長さ、ネストの深さ）
- DRY原則（重複コードの有無）
- 不要なコード（デバッグ用コード、コメントアウトされたコード）の残存

#### ロジック・正確性
- ビジネスロジックの正しさ
- エッジケース・境界値の考慮
- エラーハンドリングの適切さ
- null/undefined の安全な処理

#### セキュリティ
- SQLインジェクション対策（プレースホルダの使用）
- XSS対策（出力のエスケープ）
- 認証・認可の適切な実装
- 入力バリデーションの網羅性
- 機密情報（APIキー、パスワード等）のハードコード有無

#### パフォーマンス
- N+1クエリ問題
- 不要なクエリ・ループ処理
- インデックスが必要なカラムへの考慮
- 大量データ処理時の考慮（ページネーション、チャンク処理）

#### テスト
- 新規・変更コードに対するテストの有無
- テストケースの網羅性（正常系・異常系）
- テストの可読性と保守性

### 1-3. Claude Codeレビュー結果の出力

フェーズ1の結果をまず出力する（出力形式は後述の「出力形式」セクションに従う）。

## フェーズ2: Codexによるレビュー

Claude Codeのレビュー結果を出力した後、`mcp__codex__codex` ツールを使ってCodexにもレビューを依頼する。

### 2-1. Codexへのレビュー依頼

以下のパラメータで `mcp__codex__codex` を呼び出す:

- **prompt**: PRのレビュー依頼（リポジトリ名、PR番号、`gh pr view` と `gh pr diff` コマンドの実行指示、レビュー観点を含む）
- **sandbox**: `read-only`
- **approval-policy**: `on-failure`

promptの例:
```
GitHub PR #{PR番号} のコードレビューを実施してください。

リポジトリ: {owner}/{repo}
PR番号: {PR番号}

手順:
1. `gh pr view {PR番号} --repo {owner}/{repo} --json title,body,baseRefName,headRefName` でPR概要を取得
2. `gh pr diff {PR番号} --repo {owner}/{repo}` で差分を取得
3. 差分を以下の観点でレビュー:
   - コード品質（命名、可読性、DRY原則）
   - ロジック・正確性（ビジネスロジック、エッジケース、エラーハンドリング）
   - セキュリティ（SQLインジェクション、XSS、認証・認可）
   - パフォーマンス（N+1問題、不要なクエリ）
4. 重要度別に分類して出力（CRITICAL / WARNING / SUGGESTION / GOOD）

重要: GitHub PRへのコメント投稿は絶対に行わないこと。結果はテキスト出力のみ。
重要: 結果は日本語で出力すること。
```

### 2-2. Codexレビュー結果の出力

Codexから返された結果を「Codexによるレビュー結果」として出力する。

## フェーズ3: 統合サマリー

両方のレビュー結果を踏まえ、最後に統合サマリーを出力する:
- Claude CodeとCodexの両方で共通して指摘された項目（重要度が高い）
- 片方のみが指摘した項目（別視点からの発見）
- 総合的なマージ判断の推奨

## 出力形式

### PR概要
PRのタイトル、概要、変更ファイル数を簡潔にまとめる。

### Claude Codeレビュー結果
ファイルごとに指摘をグルーピングし、以下の重要度で分類する:

- **CRITICAL**: マージ前に必ず修正すべき問題（バグ、セキュリティ脆弱性）
- **WARNING**: 修正を強く推奨する事項（パフォーマンス問題、潜在的バグ）
- **SUGGESTION**: 改善提案（リファクタリング、可読性向上）
- **GOOD**: 良い実装として評価できる点

各指摘には以下を含める:
- 該当ファイルと行番号
- 問題の説明
- 具体的な改善案（コード例があれば提示）

### Codexレビュー結果
Codexから返された結果をそのまま出力する。

### 統合サマリー
- 共通指摘事項
- Claude Code固有の指摘
- Codex固有の指摘
- マージ可否の推奨

## 重要な制約

- **GitHub PRへのコメント投稿は絶対に行わないこと。** レビュー結果はCLI上への出力のみとする。
- `add_comment_to_pending_review`、`pull_request_review_write`、`add_issue_comment` 等のGitHub書き込み系ツールは使用禁止。
- レビューは日本語で実施する。
- Codex MCP serverへの依頼時も、GitHub書き込みを行わないよう明示的に指示すること。
